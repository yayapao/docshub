#### Debugger Nest.js App

VSCode 支持 N 种方式进行 debug

- 设置 `Debug: Toggle Auto Attach` 为 `smart`，当每次打开 vscode 的 ternimal 时，会自动进入 debug 模式
- 通过 `Debug: Create JavaScript Debug Terminal` 创建一个命令行窗口，在其内部执行命令，从而进行 debug
- 配置 `launch.json` 来编写 debug 脚本，在 vscode debug 模块内启动

##### Attach vs Launch

Attach: 依附当前进程

Launch: 另起一个新的进程

### Git

Git 相比不会陌生，这里记录安装之后的使用技巧。

全局设置用户名和账号，如果需要区分公司/个人，则在本地项目内更改设置即可（`--local`）

```bash

git config --global user.name "Your Name"
git config --global user.email "you@your-domain.com"

```

避免 `no upstream branch` 的报错

```bash

git config --global --add push.default current
git config --global --add push.autoSetupRemote true

```

修改全局 `~/.gitignore` 避免项目内重复修改，尤其是处理开源项目

```

*~
.DS_Store
.idea

```

### Charles

[charles](https://www.charlesproxy.com/) 是一个支持 HTTP 代理、监控以及反向代理的抓包工具。基于它，我们可以在线对本地脚本进行测试等。

1. 替换线上脚本，执行本地脚本，并抓取特定结果？

设置 charles 客户端：

- Proxy -> Start SSL Proxying
- Help -> SSL Proxying -> Install Charles Root Ceritificate -> 信任所有
- Proxy -> macOS proxy
- Proxy -> Proxy Setting -> enable HTTP Proxy Mode

代理指定文件:

- 过滤后选取指定文件，右键 Map local，选择指定文件进行替换
- 避免跨域问题，在拦截后 response 内添加 `Access-Control-Allow-Origin: *`
