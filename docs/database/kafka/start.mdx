## Kafka in a Nutshell

我将在该节总结 Kafka 的术语、基础概念以及运行原理。

对于 Kafka 我们需要掌握以下核心概念：

- topic
- partition
- broker

Topic: topic 作为 Kafka 内的顶级概念，无论是生产还是消费都需要指定 topic，一个 tipic 能够代表一个消息队列。

Partition: Kafka 会从逻辑上将一个 topic 分成多个 partition，每次写入 topic 内的 message 会按照策略被分配到一个 partition，每个 partition 内维护 message 的唯一 ID

Broker: Kafka 作为一个分布式集群服务，broker 是每个集群的的节点（物理机/集群），也是 message 最终写入磁盘的位置，为了保证高可用，需要将多个 partition 写到不同的 broker 上。
