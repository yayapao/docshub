:::info

React Native 可以看成是 React 在 ios/android 运行环境的实现

:::

众所周知，React 通过建立内存 DOM，即虚拟数来进行 Diff 比较、计算后，调用浏览器 API 进行重新渲染。那么只要我们能够掌握不同运行时或者执行环境下的底层渲染接口，这一套逻辑仍然行之有效。

React Native 正是利用桥接和虚拟 DOM 来在手机端内进行，**它与 React 的区别？**

- 元素标签
- 生命周期

**React Native 相比 webview 的优势？**

- webview 存在性能问题
- webview 存在样式和交互问题
- webview 维护成本更高
