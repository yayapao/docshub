本章节主要针对 [tsconfig.json](https://www.typescriptlang.org/tsconfig#module) 的使用和配置进行说明。

## How to use?

如果你的目录内含有 `tsconfig.json`，则表示改项目为一个 ts 项目，tsconfig.json 用来配置 ts 项目

```bash
# 初始化一个 tsconfig.json
tsc --init

## 基于 tsconfig.json 进行打包
tsc --build tsconfig.json
```

## Configuration

让我们从一份通用配置开始：

```js
{
  "compilerOptions": {
    "outDir": "dist",
    "module": "esnext", // 编译后输出的模块，决定代码的组织形式
    "target": "esnext", // 编译后，输出代码的 ES 版本，决定代码的语法规范
    "lib": ["esnext", "dom"],
    "sourceMap": true, // 生成 sourcMap 文件
    "declaration": true, // 生成 .d.ts 的声明文件
    "declarationMap": false, // 基于 .d.ts 生成 .d.ts.map 文件
    "baseUrl": ".",
    "jsx": "react",
    "allowSyntheticDefaultImports": true,
    "moduleResolution": "node",
    "forceConsistentCasingInFileNames": true,
    "noImplicitReturns": true,
    "suppressImplicitAnyIndexErrors": true,
    "noUnusedLocals": true,
    "allowJs": true,
    "skipLibCheck": true,
    "experimentalDecorators": true,
    "strict": true,
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "exclude": ["node_modules", "build", "dist", "scripts", "webpack", "jest"]
}
```
