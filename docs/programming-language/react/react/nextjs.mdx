## 文件组织结构

Next.js 按照约定，使用声明式的方式来组织文件目录，这样可以让你的项目更加清晰。

其中亮点之一是**通过文档结构来组织路由，并且能够处理特殊的路由情况。**也就是说，在 Next.js 的设计规则下，你必须遵循它的设计规范来组织文档结构。

![Page-Structure](https://nextjs.org/_next/image?url=%2Fdocs%2Fdark%2Fdefining-routes.png&w=3840&q=75&dpl=dpl_L5sjBhALQ2CCSYjrGLdXzzBWG9An)

:::warning

Next.js 内支持 js, jsx 以及 tsx，我们默认使用 tsx 作为文件后缀。

:::

一个页面内容包括：`page.tsx(required)`, `layout.tsx`, `loading.tsx`, `error.tsx`, `default.tsx`

- 在目录内，使用 `page.tsx` 来定义路由对应的**主页面**，使用 `layout.tsx` 来定义**组件布局**，布局支持嵌套
- 顶层的 `layout.tsx` 文件内必须包含 `html` 和 `body`，它被称为 Root Layout，**Root Layout 是必须的**
- 除了 `layout` 之外，还存在另一种选择 -- `template`，`template` 不会保留 state 并且每次会重新渲染页面

`template.js` 在 layout 和 children 之间渲染，如下：

```tsx
<Layout>
  {/* Note that the template is given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```
