在了解谷歌定义的 Web 指标后，我们迫不及待想采集这些指标，看看应用性能如何。对于指标采集，我们需要注意 ⚠️，尽可能使用浏览器 API，而不是一些兼容技巧，避免这些技巧本身产生副作用（比如增大耗电等）。

:::caution 原则

有效的性能测量的第一条规则是确保性能测量技术本身不会导致性能问题

:::

指标采集应当分为两个模块：Web Metrcis 和自定义指标。社区内已经有一些成熟的解决方案：

- [wpmonit](https://github.com/Y-lonelY/wpmonit)
- [GoogleChrome/web-vitals](https://github.com/GoogleChrome/web-vitals)
- [perfume](https://github.com/Zizzamia/perfume.js)
- [sentry-javascript](https://github.com/getsentry/sentry-javascript)

因此本篇将围绕浏览器的原生支持和测量工具来展开。
