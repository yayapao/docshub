## Core Web Vitals

对于核心指标，Google 提出了三个最核心的指标：LCP、FID 和 CLS，分别对应加载性能、可交互性以及视觉稳定性。

<HighlightWithText>如何理解核心?</HighlightWithText>

- 这三个指标不仅能够被实际测量，并且适用于所有网页应用，更关键的是，它们能够反映**以用户为中心**的关键结果的真实体验，即 RUM 指标。

⚠️ 注意：各个指标的衡量指标具有广泛的指导意义，但是对于特殊的应用其阈值可以动态调整

### LCP(Largest Contentful Paint)

[LCP](https://web.dev/lcp/) 用来衡量页面加载性能

对于为什么选用最大绘制内容作为衡量标准，很大程度是因为其他指标不够准确，比如当进入页面先加载一个 loading 效果，即使主要内容还没加载出来，此时页面会认为已经完成了 load 、DOMContentLoaded 或者被捕获到 FCP 指标。因为 Google 选择使用 LCP 来作为 RUM 加载性能的衡量标准，为此也给了一个经典的解释：

> "Sometimes simpler is better. A more accurate way to measure when the main content of a page is loaded is to look at when the largest element was rendered!"

衡量标准：75%的访问用户其真实 LCP 小于 2.5s，则认为该应用表现良好，大于 4s 则表现糟糕
